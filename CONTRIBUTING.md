# Руководство по участию в разработке

## Установка

1. Клонируйте репозиторий
2. Установите зависимости: `npm run install:all`
3. Настройте `.env` файлы
4. Запустите базу данных
5. Запустите миграции: `cd backend && npm run migrate`
6. Запустите проект: `npm run dev`

## Структура коммитов

Используйте следующие префиксы:
- `feat:` - новая функциональность
- `fix:` - исправление бага
- `docs:` - документация
- `style:` - форматирование кода
- `refactor:` - рефакторинг
- `test:` - добавление тестов
- `chore:` - обновление зависимостей и т.д.

## Рабочий процесс

1. Создайте ветку от `main`
2. Внесите изменения
3. Протестируйте локально
4. Создайте Pull Request
5. Дождитесь review

## Код-стайл

- Используйте ESLint и Prettier
- Следуйте TypeScript best practices
- Покрывайте новый код тестами
- Пишите документацию для новых функций

## Вопросы

Создайте Issue для вопросов и обсуждений.
